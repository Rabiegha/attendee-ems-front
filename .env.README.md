## üìÅ File Structure

```
.env.example     ‚Üê Template (committed to Git)
```

## üöÄ Quick Start

### 1. Local Development

```bash
cp .env.example .env.local
```

Edit `.env.local`:
```env
VITE_API_BASE_URL=http://localhost:3000
VITE_SENTRY_DSN=
VITE_APP_VERSION=1.0.0
```

Then run:
```bash
npm run dev
```

### 2. VPS Staging Deployment

```bash
cp .env.example .env.vps
```

Edit `.env.vps`:
```env
VITE_API_BASE_URL=https://api.attendee.fr
VITE_SENTRY_DSN=https://YOUR_DSN@sentry.io/PROJECT_ID
VITE_APP_VERSION=1.0.0
```

Build for VPS:
```bash
npm run build:vps
```

### 3. Google Cloud Production

```bash
cp .env.example .env.gcloud
```

Edit `.env.gcloud`:
```env
VITE_API_BASE_URL=https://api-prod.attendee.fr
VITE_SENTRY_DSN=
VITE_APP_VERSION=1.0.0
```

Build for GCloud:
```bash
npm run build:gcloud
```

## üîë Configuration Variables

### Required

- **VITE_API_BASE_URL**: Backend API URL
- **VITE_APP_VERSION**: Application version

### Optional

- **VITE_SENTRY_DSN**: Sentry error tracking (leave empty to disable)
- **VITE_GOOGLE_MAPS_API_KEY**: Google Maps integration

1. ‚úÖ **DO**: Keep `.env.local`, `.env.vps`, `.env.gcloud` on your machine only
2. ‚úÖ **DO**: Use different credentials for each environment
3. ‚úÖ **DO**: Rotate secrets regularly

## üõ†Ô∏è Troubleshooting

### "Variable not defined" error

Make sure you:
1. Created the `.env.local` file (not just `.env`)
2. Restarted the dev server after changes
3. Variables start with `VITE_` prefix

### Build uses wrong environment

Check that you're using the correct command:
- `npm run build:vps` for VPS
- `npm run build:gcloud` for GCloud
